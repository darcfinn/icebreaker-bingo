# Project Structure Documentation

## Overview
Modern web application built with Vite, using Tailwind CSS for styling and Firebase for backend services.

## Root Level Files
- `package.json` - Dependencies and npm scripts
- `package-lock.json` - Locked dependency versions
- `vite.config.js` - Vite bundler configuration
- `tailwind.config.js` - Tailwind CSS customization
- `postcss.config.js` - PostCSS plugins (Tailwind processing)
- `index.html` - Entry HTML file (Vite serves from root)
- `test-firebase.html` - Firebase integration testing

## Directory Structure

### `/src` - Application Source Code
Primary application code directory.

#### `/src/main.js` or `/src/main.ts`
Application entry point. Initializes app, sets up routing, global providers.

#### `/src/components`
Reusable UI components following atomic design principles.

```
/components
  /ui              # Base UI elements (buttons, inputs, cards)
    Button.jsx
    Input.jsx
    Card.jsx
  /layout          # Layout components (header, footer, sidebar)
    Header.jsx
    Footer.jsx
    Sidebar.jsx
  /features        # Feature-specific components
    /auth
      LoginForm.jsx
      SignupForm.jsx
    /dashboard
      DashboardWidget.jsx
```

**Naming convention**: 
- PascalCase for components (e.g., `UserProfile.jsx`)
- Descriptive constant and variable names, minimum 5 characters

#### `/src/pages` or `/src/views`
Route-level components (each represents a page/view).

```
/pages
  Home.jsx
  About.jsx
  Dashboard.jsx
  NotFound.jsx
```

**Naming convention**: PascalCase, typically matches route name

#### `/src/hooks`
Custom React hooks for reusable logic.

```
/hooks
  useAuth.js       # Authentication state/logic
  useFirebase.js   # Firebase operations
  useLocalStorage.js
  useDebounce.js
```

**Naming convention**: camelCase starting with 'use'

#### `/src/utils` or `/src/helpers`
Pure utility functions, helpers, formatters.

```
/utils
  dateFormatter.js
  validators.js
  api.js           # API call helpers
  constants.js
```

**Naming convention**: camelCase for files and functions

#### `/src/services`
Business logic layer, API integrations, external service wrappers.

```
/services
  authService.js
  firebaseService.js
  apiService.js
```

**Purpose**: Abstracts external dependencies from components

#### `/src/store` or `/src/context`
State management (Context API, Zustand, Redux, etc.).

```
/store
  authStore.js
  userStore.js
  
# OR for Context API
/context
  AuthContext.jsx
  ThemeContext.jsx
```

#### `/src/styles`
Global styles, CSS modules, theme definitions.

```
/styles
  global.css       # Global styles, Tailwind imports
  variables.css    # CSS custom properties
  themes/          # Theme configurations
```

#### `/src/assets`
Static assets (images, fonts, icons).

```
/assets
  /images
    logo.svg
    hero.jpg
  /icons
  /fonts
```

#### `/src/config`
Configuration files, environment variable management.

```
/config
  firebase.js      # Firebase configuration
  routes.js        # Route definitions
  app.config.js    # App-wide settings
```

#### `/src/lib`
Third-party library configurations and wrappers.

```
/lib
  firebase.js      # Initialized Firebase instance
  axios.js         # Configured Axios instance
```

#### `/src/types` (if using TypeScript)
Type definitions and interfaces.

```
/types
  user.ts
  api.ts
  index.ts         # Re-export all types
```

### `/dist`
Production build output (generated by Vite, not committed to Git).

### `/node_modules`
NPM dependencies (not committed to Git).

### `/public` (optional)
Static files served as-is (not processed by Vite).
- Favicons
- robots.txt
- Static assets that don't need processing

## Code Organization Principles

### Component Structure
```jsx
// Imports (external, then internal)
import React, { useState } from 'react';
import { Button } from '@/components/ui';

// Component definition
export default function MyComponent({ prop1, prop2 }) {
  // Hooks
  const [state, setState] = useState();
  
  // Handlers
  const handleClick = () => {};
  
  // Render
  return <div>...</div>;
}
```

### File Naming Conventions
- **Components**: PascalCase (`UserProfile.jsx`)
- **Hooks**: camelCase with 'use' prefix (`useAuth.js`)
- **Utils/Services**: camelCase (`authService.js`)
- **Constants**: UPPER_SNAKE_CASE or camelCase file with UPPER exports

### Import Path Aliases (configure in vite.config.js)
```javascript
'@/components' → '/src/components'
'@/hooks' → '/src/hooks'
'@/utils' → '/src/utils'
'@/services' → '/src/services'
```


## Development Workflow
1. **Feature branches**: Create from `main` for new features
2. **Component development**: Build in isolation, test independently
3. **Service layer**: All Firebase/API calls go through service layer
4. **Environment variables**: Use `.env` for Firebase keys (never commit)

## Firebase Integration
- Initialize Firebase in `/src/lib/firebase.js`
- Wrap operations in `/src/services/firebaseService.js`
- Use custom hooks for Firebase state (`useAuth`, `useFirestore`)

## Testing Structure (recommended addition)
```
/src
  /__tests__       # Test files
    /components
    /hooks
    /utils
  /[feature].test.js  # Co-located tests
```

## Notes
- Keep components small and focused (single responsibility)
- Separate business logic from presentation (use hooks/services)
- Components should receive data via props, not fetch directly
- Use TypeScript for better type safety (recommended migration)
